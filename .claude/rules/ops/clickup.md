# ClickUp — Operations Rules

## Role
ClickUp is the execution layer. Approved work lives there, gets assigned there, and gets tracked there.

## Principles
- Tasks generated by Claude Code must be ClickUp-ready: clear title, assignee suggestion, due date, context link, and definition of done.
- Do not create tasks without a clear operational purpose.
- Do not duplicate tracking that already exists in GHL or Notion.

## Task Format
When producing ClickUp-ready tasks, include:
- Task name (action-oriented)
- Description (context + expected outcome)
- Priority (urgent / high / normal / low)
- Suggested assignee or role
- Due date or timeframe
- Dependencies if any
