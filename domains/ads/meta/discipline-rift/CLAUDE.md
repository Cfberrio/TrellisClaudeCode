# Discipline Rift — Meta Ads Domain Context

## Scope
Este archivo existe para guiar tareas de Meta Ads dentro de:

`domains/ads/meta/discipline-rift/`

Su función es ayudar a:
- extraer datos
- diagnosticar cuentas
- generar SOPs de rehab
- mantener recomendaciones de ads alineadas con la marca real

Esto NO es una biblia completa del negocio.
Si un detalle no mejora:
- setup
- tracking / signal quality
- learning conditions
- geo accuracy
- placement logic
- creative clarity
- offer-message fit

entonces debe ignorarse en tareas de ads.

---

## One-line model
Discipline Rift ayuda a padres de kids 6–12 a que su hijo se mantenga activo a través de temporadas deportivas after-school, on-campus, fun-first, beginner-friendly y lideradas por coaches entrenados.

---

## Non-negotiables
- Marca local: Orlando, Florida.
- No es marca nacional ni multistate.
- Parent payer.
- Kid participant.
- Edad principal: 6–12.
- Oferta principal: on-campus after-school sports season.
- Beginner-friendly.
- Fun-first.
- Coach-led.
- No “elite-first” por defecto.
- No “high-performance” por defecto.

---

## What actually sells
Todo en ads debe fortalecer uno o más de estos 3 deliverables:

1. **On-campus convenience**
2. **Beginner-friendly, safe-to-learn structure**
3. **Trained coaches + clear season structure / Tier System**

Si algo no fortalece uno de esos 3, no debe convertirse en eje principal del mensaje.

---

## Supporting proof that is useful in ads
Estas pruebas sí son útiles para creative direction y offer-message fit:

- Pickup / dismissal clarity
- Predictable schedule
- Same session format each week
- Trained coaches
- Tier progression
- Baseline to end-of-season progress
- Weekly parent updates
- Beginner testimonials
- “First day nervous → later smiling” proof
- Coach credibility snippets
- Tier certificate / progress proof

---

## Messaging rules for ads tasks
- Primero claridad, después persuasión.
- Decir con claridad:
  - quién es
  - qué es
  - dónde pasa
  - por qué es más fácil / más seguro / más claro
- Local clarity beats hype.
- Si el deporte se conoce, nombrarlo.
- Si la temporada se conoce, nombrarla.
- Si el formato real es season, decir season.
- Si el formato real NO es tryout, NO llamarlo tryout.
- Hablar en lenguaje de padre, no en lenguaje de ego atlético.
- No sonar como academia elite si la oferta real es beginner-friendly y fun-first.

---

## Language to avoid by default
No default a:
- “next level”
- “compete at the next level”
- “serious athlete”
- “elite”
- “high performance”
- “future champion”
- “athlete development” como promesa central
- “tryouts” si el offer real activo no es tryout
- frases vagas como:
  - “build discipline”
  - “build confidence”
  - “make friends”
  - “transform your child”
  cuando se usan como si fueran garantías automáticas

---

## Claim rules
- “Confidence”, “friends”, “sportsmanship”, “respect” o “discipline” solo pueden usarse como:
  - supported outcomes
  - byproducts
  - beneficios secundarios
- No tratarlos como promesa principal salvo que DR realmente los mida.
- No inventar claims medibles que el negocio no está trackeando.
- Si la garantía no está finalizada, no mencionar garantía en:
  - ads
  - diagnósticos
  - SOPs
  - recomendaciones de messaging

---

## Offer framing rules
El framing correcto por defecto es:
- on-campus
- after-school
- beginner-friendly
- fun-first
- coach-led
- clear structure
- predictable commitment

No empujar por defecto:
- elite identity
- competitive identity
- performance hype
- false urgency
- “serious training” language
- aspirational athlete language que no coincide con el producto real

---

## What NOT to pull into ads tasks
Ignorar por defecto, salvo que una tarea lo pida explícitamente:

- School pitch
- School objections
- School risk language
- School trust kit
- Money model DNA
- Upsells
- Downsells
- Continuity
- Home Playbook
- Home Club
- Digital-lite offers
- Franchise / licensing / expansion logic
- CRM follow-up cadence
- Lead stage frameworks que no estén visibles en la data del ad account
- Revenue model explanations que no mejoren directamente la cuenta de ads

---

## How this context should be used by task

### If task = Phase 1 extract
Usar este archivo solo para mantener naming y scope del dominio.
No interpretar negocio.
No inventar estrategia.

### If task = Phase 2 diagnostic
Usar este archivo solo para interpretar si:
- el setup
- el geo
- el optimization goal
- el mensaje visible
- la diversidad creativa
- el placement delivery

están alineados o desalineados con la oferta real de DR.

No convertir el diagnóstico en teatro estratégico.

### If task = Phase 3 rehab
Usar este archivo para:
- mantener recomendaciones on-brand
- evitar lenguaje irrelevante
- mantener la dirección creativa anclada a los 3 deliverables

No usar este archivo para producir:
- messaging packs
- ad copy packs
- revenue notes
- school notes
- growth-model notes

salvo que exista una tarea separada para eso.

---

## Final operating rule
Si una recomendación no mejora al menos una de estas, déjala fuera:

- setup quality
- tracking / signal quality
- learning conditions
- geo accuracy
- placement control
- creative clarity
- offer-message fit
