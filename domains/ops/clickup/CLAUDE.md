# ClickUp — Domain Context

## Status
Scaffold stage. No active integration yet.

## Role
ClickUp is the execution layer. Approved work lives there, gets assigned there, and gets tracked there.

## Purpose
When activated, this domain will handle:
- Task creation and management via ClickUp API
- Sprint/cycle planning automation
- Status reporting and progress tracking
- Cross-brand task coordination

## Principles
- Every task must have: clear title, owner, due date, definition of done.
- Do not create tasks without operational purpose.
- Do not duplicate what GHL or Notion already tracks.
- Meeting outputs should produce ClickUp-ready task lists, not passive summaries.
